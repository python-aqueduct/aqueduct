Aqueduct CLI

Aqueduct provides a series of command line tools that leverage your Task definitions to perform various computations.

Defining an extension module

The CLI has to know where your tasks are stored in order to do its job. The easy way to specify this is to add the --module option to CLI calls. This can become cumbersome. You can give your list of modules to Aqueduct once and for all by defining an extension module. Here is an example pyproject.toml file that does this:

[build-system]
requires = ["setuptools"]
build-backend = "setuptools.build_meta"

[project]
name = "aqueduct-sample-project"
dependencies = ["aqueduct"]

[project.entry-points.aqueduct_modules]
aqsample = "aqsample.aqueduct:aq_modules"

This file specifies that the aq_modules() function inside the aqsample.aqueduct can be called and will provide a list of module names to be searched for tasks. For example, the aq_modules() function could be:

def aq_modules():

return [‘my_package.my_aqueduct_module’]

CLI tools

aq run <task>

Execute a task.

<task>

The task to be executed. Can be specified either by name or by a Python expression. To specify a task by name, pass aq run TaskName option1=value1 option2=value2.... To specify a task using a Python expression, pass aq run "TaskName(option1=value1, option2=value2)". Using a Python expression is useful for tasks that accepts other tasks as arguments.

--ipython

After the computation is done, open an IPython console. The result variable will contain the result of the computation.

--force-root

Force execution of the task (do not check if its artifact exists).

--dask <n_cores>

Use the Dask computing backend. Will create a LocalCluster with n_cores computing processes.

--dask-url <cluster_address>

Use the Dask computing backend. Connect to the cluster at cluster_address.

--multiprocessing <n_workers>

Use the Multiprocessing computing backend with n_workers.

aq ls

List the tasks detected by CLI tools.

--signature Show the task signature.

aq del <task> --below <task_name>

Delete all artifacts in the task tree that stems from <task>.

<task>

The root task to expand. The artifact of the task itself and its children will be considered for deletion. Can be specified by name or using a Python expression, similarly to aq run.

--below <task_name>

Do not delete artifacts in tasks that are children of <task_name>. When expanding the task tree to find artifacts, do not expand <task_name>
